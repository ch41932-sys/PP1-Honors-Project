Apex Drive Customizer Programming Project Reflection
The “Apex Drive Customizer” is a Java-based application that I developed to stimulate configuring a high-tech luxury SUV. My primary objective in this project was to apply foundational concepts from Programming Principles I—such as object oriented design, data organization, and exception handling—in a cohesive and professional manner. By going beyond standard course assignments, I aimed to deepen my understanding of program design and documentation, making this project a meaningful learning experience. Through the development of this project, I significantly strengthened my understanding of objects and classes. Creating separate classes for the SUV and Upgrade required me to break down the problem into logical components, reinforcing the importance of modularity in software design. I emphasized proper encapsulation by using private fields and public methods, ensuring the integrity and security of the program’s data. Additionally, I integrated ArrayLists to organize data effectively. Unlike standard arrays, ArrayLists to allowed my program to dynamically store any number of user-selected upgrades. This demonstrated to me how appropriate data structures can greatly increase a program’s flexibility and scalability. One of the primary challenges I encountered was ensuring that my modular design functioned correctly across multiple files. Initially, I faced errors because the Main class could not resolve the other custom classes. I addressed this issue by organizing my project into a logical folder structure within my IDE and ensuring that all files resided in the same package.
Another challenge involved making the program robust enough to handle invalid user input. To overcome this, I implemented exception handling with try/catch blocks. This approach ensured that if a user entered a letter instead of a menu number, the program would display an informative error message rather than crash, thereby significantly improving the program’s reliability and user experience. This project serves as a bridge to future programming courses, such as Programming Principles II and Data Structures. Learning professional workflows using GitHub—including maintaining a detailed commit history and writing a comprehensive README—has equipped me for the collaborative environments I will encounter in advanced computer science courses and in the software industry. Given more time to refine this project, I would implement text to allow users to save their final vehicle configuration to a file. Additionally, I would explore using 2D arrays to manage a larger inventory of vehicle models and their respective specifications in a structured, tabular format. These enhancements would further increase the program’s usability and complexity. Developing the Apex Drive Customizer has been a valuable exercise in growth, discipline, and curiosity. It allowed me to transform abstract coding concepts into a fucntional, well-documented system, marking my progress as a programmer during my first year at Georgia Southern. This experience has laid a solid foundation for my continued studies in computer science and my future endeavors in software development.
